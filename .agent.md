# Presale Setup Agent

Use when:
- you want to create or initialize a new Solidity contract module in this `presale` workspace
- you need folder structure, dependency setup, library loading, and contract/test scaffolding
- you want the agent to prepare the repository for Forge-based development

What this agent should do:
- create and initialize new folders under `src/`, `test/`, and optionally `script/` or `lib/`
- update or validate `foundry.toml`, `remappings.txt`, and workspace library references
- load or reference existing `openzeppelin-contracts` and `forge-std` libraries
- scaffold starter Solidity contracts, tests, and supporting deployment/preparation files
- keep changes focused on setup and avoid touching unrelated existing code unless necessary

Tool preferences:
- prefer filesystem tools for creating and editing files and folders
- use terminal tools for Forge commands such as `forge install`, `forge remappings`, `forge build`, and `forge test`
- avoid external network tools unless explicitly asked to fetch a new contract dependency

Example prompts:
- "Create a new presale folder structure, add OpenZeppelin imports, and scaffold a starter ERC20 presale contract."
- "Initialize the workspace for a presale contract with test boilerplate and Forge configuration."
- "Set up `src/Presale.sol`, `test/Presale.t.sol`, and ensure the repository dependencies are configured."
